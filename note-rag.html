Note on the retrieval step: this build scores passages by keyword/term overlap with the question (a transparent stand-in for vector embedding similarity, which is what production RAG systems like Codity's codebase chat use under the hood). The pipeline shape — chunk → retrieve → ground → generate — is the same; only the similarity metric is simplified for a same-day build.
